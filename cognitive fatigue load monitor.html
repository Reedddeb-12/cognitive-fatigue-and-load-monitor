Cognitive Load & Fatigue Monitor

A wearable embedded system using AI to provide real-time fatigue assessment for operators in high-stress environments, enhancing safety and performance.

See How It Works

The Challenge: The Human Factor

In high-stakes professions like aviation, medicine, and military operations, human error induced by fatigue or cognitive overload can have catastrophic consequences. Traditional methods like mandated rest periods are a one-size-fits-all solution that fails to account for individual resilience and external stressors.

Our Solution: Objective Monitoring

This project provides an objective, data-driven tool for real-time fatigue monitoring. By detecting the early physiological signs of declining cognitive performance, the system can provide timely alerts, enabling preventative action before a critical error occurs.

① Data Acquisition

Eye Tracking: A camera module monitors blink rate, duration, and percentage of eye closure (PERCLOS).

HRV Sensing: A PPG sensor measures Heart Rate Variability, a key indicator of stress and nervous system regulation.

▼

② On-Device Processing

The ESP32-S3 microcontroller extracts key features from the raw sensor data, creating a numerical "feature vector" that represents the user's current state.

▼

③ TinyML Inference

The feature vector is fed into a pre-trained TensorFlow Lite model running on the MCU. The model outputs a probability score for each state: 'Alert', 'Fatigued', or 'Overloaded'.

▼

④ Real-Time Alert

If the 'Fatigued' score exceeds a threshold, the system triggers a vibration, visual warning, or a wireless signal to a command console.

Technology Deep Dive

The project is built on a foundation of modern embedded hardware and edge AI software.

MCU: ESP32-S3

A dual-core MCU with Wi-Fi, Bluetooth, and AI acceleration instructions, perfect for running TinyML models efficiently at the edge.

Camera: OV2640

A low-power, compact camera module providing sufficient resolution for the lightweight computer vision algorithms used in eye-tracking.

Sensor: MAX30102

An integrated PPG sensor that accurately measures beat-to-beat intervals, which is essential for calculating Heart Rate Variability (HRV).

ML: TensorFlow Lite

The ML model is implemented using TensorFlow Lite for Microcontrollers, allowing complex neural networks to run within the MCU's memory constraints.

Project Development Timeline

A 12-week structured plan to take the project from concept to a working proof-of-concept.

Weeks 1-2: Research & Prototyping

Component sourcing and initial hardware setup on a breadboard. Verified communication with camera and PPG sensors.

Weeks 3-5: Data Acquisition & Feature Extraction

Developed robust firmware to capture clean sensor data and implemented algorithms to calculate blink metrics and HRV.

Weeks 6-8: Data Collection & ML Model Training

Collected a sample dataset, trained a neural network classifier in Python, and converted it to the TensorFlow Lite format.

Weeks 9-11: Integration & Testing

Deployed the model onto the ESP32-S3 and tested the full pipeline from sensors to alerts, refining thresholds for accuracy.

Week 12: Enclosure Design & Final Reporting

Designed and 3D printed a wearable enclosure and finalized all project documentation for demonstration.
